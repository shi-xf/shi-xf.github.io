4.事件流

事件流：描绘事件在文档中传递顺序。（谁先接受事件，谁后接受事件） 捕获事件流：事件从不具体元素（window）向具体的元素传递（inner）。事件从父元素传递到子元素 元素.addEventListener("click",fn1,true) 第三个参数：事件会在捕获阶段触发（true） 冒泡事件流：和捕获事件流相反，事件从具体元素（inner）向不具体元素（window）传递。事件从子元素传递到父元素 元素.addEventListener("click",fn1,true )第三个参数：事件和捕获事件流相反（默认:false） ### 5.DOM2事件流：事件传递分为3个阶段，①事件捕获阶段②目标元素上停留③事件冒泡阶段 ### 6.阻止事件冒泡 标准浏览器使用 var ev = event || e; e代表函数传入的第一个参数 ev.stopPropagetion() 低版本ie：ev.cancelBubble = true; ### 7.阻止默认事件 标准浏览器使用 ev.preventDefault(); DOM 零级阻止：return false ie低版本：ev.returnValue = false; ### 8.键盘事件 onkeydown 按键按下去触发的事件 document.onkeydown = function(e){ var ev = event || e; ev.keyCode 获取案件的键码值 } onkeyup 按键抬起 onkeypress 按键按压 ### 9.滚轮事件 标准chrome,ie onmousewheel document.onmousewheel = function(e){ var ev = event || e; ev.wheelDelta 向上滚为120，向下滚为-120 } 火狐 document.onmousewheel = fn1; document.addEventListener("DOMMouseScroll",fn1); function fn1(e){ var ev = event || e; ev.detail 向上滚为-3，向下滚为+3 } ### 10.事件委托 事件委托（事件代理）：元素将事件委托给父元素来处理（通过事件冒泡） 父元素对事件源对象进行甄别，然后进行对应的处理 好处：①可以给后续添加的元素绑定事件 ②可以优化性能
